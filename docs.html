타이타닉 생존자 예측 프로젝트 보고서

Kaggle Titanic 문제를 풀며 점수를 올린 과정을 정리한 보고서입니다.

완료

프로젝트 개요

머신러닝을 직접 해보고 싶어서 Kaggle의 Titanic 문제를 선택했다. 승객의 성별, 나이, 객실 등급, 요금 같은 정보를 보고 생존 여부를 예측하는 문제였다. 처음에는 모델을 돌려 점수만 올리면 되는 줄 알았지만, 실제로는 어떤 정보가 의미 있는지 고르는 과정이 더 중요했다.

데이터 확인

train.csv에는 891명의 정보와 생존 여부가 있었고, test.csv에는 418명의 승객 정보만 있었다. 데이터를 살펴보니 성별과 객실 등급이 가장 눈에 띄었다. 여성의 생존율은 남성보다 훨씬 높았고, 1등석 승객도 3등석 승객보다 생존율이 높았다.

처음 시도한 방법

처음에는 RandomForest와 GradientBoosting 같은 모델을 사용했다. 이름에서 Mr, Mrs, Miss, Master 같은 호칭을 뽑고, 가족 수를 계산하고, 비어 있는 나이는 호칭별 중앙값으로 채웠다. 교차검증에서는 GradientBoosting이 0.8485 정도까지 나왔지만, 실제 제출에서는 0.755 정도로 떨어졌다.

점수를 올린 핵심 방법

이후에는 모델을 더 복잡하게 만드는 대신 가족 그룹 규칙을 적용했다. 외부에서 정답을 찾아온 것이 아니라 train 데이터의 라벨만 사용했다. 기본적으로 여성과 어린 남자아이로 볼 수 있는 Master는 생존, 성인 남성은 사망으로 두고 시작했다. 그다음 같은 성과 같은 티켓을 가진 사람들을 하나의 가족 그룹으로 묶었다.

train 데이터에서 어떤 가족의 여자와 아이가 전부 사망했다면, test 데이터에 같은 가족으로 보이는 여자와 아이가 있을 때도 사망으로 바꿨다. 반대로 어떤 가족의 남성이 train에서 모두 생존했다면, test에 있는 같은 가족의 남성도 생존으로 바꿨다.

결과

최종 점수0.78947
순위1476등
제출 횟수3회
핵심 기법가족 그룹 규칙

처음 7000등대였던 기록에서 1476등까지 올라갔다. 0.8을 넘기지는 못했지만, 단순히 모델만 바꾼 것이 아니라 데이터 안의 관계를 이용해 점수를 올렸다는 점에서 의미가 있었다.

느낀 점

이번 프로젝트에서 가장 크게 배운 것은 복잡한 모델이 항상 좋은 것은 아니라는 점이다. CV 점수가 높아도 실제 제출에서는 점수가 낮게 나올 수 있었고, 오히려 설명 가능한 간단한 규칙이 더 안정적인 결과를 만들기도 했다.
